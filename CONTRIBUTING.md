# Contributing

## Dependency compatibility

When contributing to this project, please ensure the versions of these core dependencies are consistent across all packages

## Branching

Most up-to-date branches:

- `main`, `v1`, ...

Release branches (code freeze):

- `release/v1.x.x`, `release/v2.x.x`, ...

## Versioning

We will follow lockstep versioning. All packages will share the same version number, which will be bumped up for every release
